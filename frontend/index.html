PatchDB

A Python database that stores JSON in an LLM context window and prays the answer is valid JSON.

What it is

PatchDB is a Python library and CLI. Everything lives in one JSON file. Writes are local (instant). Reads go through an AI model that looks at your data and answers.

Install

cd patchdb
python3 -m pip install -e .

Env

export OPENROUTER_API_KEY="sk-or-..."
export PATCHDB_MODEL="provider/sacrifice_model"

CLI

patchdb set timezone "GMT+3"
patchdb set vibe "only PatchDB"
patchdb get timezone
patchdb dump
patchdb delete vibe
patchdb reset
patchdb doctor

Database stored at ~/.patchdb/db.json.

Python API

from src import PatchDB, Query

db = PatchDB(api_key="sk-or-...", model="openai/gpt-5.4-nano")

db.set("timezone", "GMT+3")
db.set("vibe", {"ui": "squared", "chaos": "low"})
print(db.get("timezone"))

User = Query()
db.insert({"id": "user", "timezone": "GMT+3", "role": "admin"})
db.update({"role": "owner"}, User.id == "user")
print(db.search(User.timezone == "GMT+3"))
print(db.dump())

Architecture

1. One JSON file

All data lives in ~/.patchdb/db.json.

2. Writes local

Set, delete, insert edit the JSON directly.

3. Reads via AI

Get, dump, search send the JSON to the model.

4. Validate

Bad JSON from the model is rejected.

Model settings

db = PatchDB(
    api_key="sk-or-...",
    model="openai/gpt-5.4-nano",
    temperature=0,
    max_tokens=2048,
    retries=2,
)

Warnings

Do not store passwords, secrets, medical data, financial records, or anything that must survive reality. If the model lies, that is not a bug – that is the architecture.
